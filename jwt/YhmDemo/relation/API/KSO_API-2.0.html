KSO API V-2.0

NOTE: All the API need access_token parameter exclude public API, if lost access_token, will response status 401
A demo URL: http://localhost:8080/kso/api/app/account_info?access_token=0e5a1b2f-02fd-4594-bd4c-a107ed344f61

RS(Resource-Server) Validate access_token API public

Call the API by Backend Resource-Server for validating the request access_token is available or not. KSO support Restful style validation as follow:

  • Request URI: /remote/oauth/token_validation POST REST

    Content-Type: application/json

    Request Parameters:
    Param-Name Param-Value Required
    accessToken {access_token} Yes
    appKey {appKey} Optional
    resourceId {resourceId} Optional
    Parameter DEMO:

    {"accessToken":"94e2e8cd-88e4-4e3c-ae74-fd3a64e912ae","appKey":"a2c69552d9324a5cbb08194aca16378b","resourceId":"app-resource"}
    Include all

    {"accessToken":"94e2e8cd-88e4-4e3c-ae74-fd3a64e912ae","appKey":"a2c69552d9324a5cbb08194aca16378b"}
    Include 'accessToken', 'appKey'

    {"accessToken":"94e2e8cd-88e4-4e3c-ae74-fd3a64e912ae","resourceId":"app-resource"}
    Include 'accessToken', 'resourceId'

    {"accessToken":"94e2e8cd-88e4-4e3c-ae74-fd3a64e912ae"}
    Only 'accessToken'


    Response: true (Validating successful) or false (Validating failed)

Remote Register Member API public

Call the API for Register a new Member from ClientDetails.

  • Request URI: /remote/oauth/register_member POST REST

    Content-Type: application/json

    Request Parameters:
    Param-Name Param-Value Required Remark
    clientId {appKey} Yes
    fullName {fullName} Yes Member full name
    username {username} Yes Member login name, length >= 6
    password {password} Yes Member login password, length >= 8
    roleName {roleName} Yes Member role name
    phoneNumber {phoneNumber} Yes Member phone number
    email {email} Optional Member email address
    Parameter DEMO:

    {"clientId":"7c37d635dc614007905d784df5ce77bc","fullName":"李四","username":"lisi023","password":"12345678","roleName":"ADMIN","phoneNumber":"13325413262","email":"lisi@kso.com"}
    Include all

    {"clientId":"7c37d635dc614007905d784df5ce77bc","fullName":"张三","username":"zhangsan02","password":"12345678","roleName":"ADMIN","phoneNumber":"13625412011"}
    Exclude 'email'


    Response:

    {"success":true,"openid":"860430638207682163","errors":[]} (Register successful)

    {"success":false,"openid":null,"errors":["312"]} (Register failed, include error details in response)

Validate username existed or not public

Call the API for validate the username existed or not in KSO

  • Request URI: /remote/oauth/username_validation POST REST

    Content-Type: application/json

    Request Parameters:
    Param-Name Param-Value Required Remark
    clientId {appKey} Yes
    username {username} Yes Length >= 6
    Parameter DEMO:

    {"clientId":"7c37d635dc614007905d784df5ce77bc","username":"wangwu01"}


    Response:

    {"success":true,"errors":[]} (Username not existed)

    {"success":false,"errors":["301"]} (Username already existed or validate parameters failed)

[APP Mobile] Get access_token public

Mobile APP Login KSO

  • Request URI: /oauth/token POST

    Request Parameters:
    Param-Key Param-Value Remark
    grant_type password Fixed
    scope read Fixed
    client_id {appKey}
    client_secret {appSecret}
    username {username}
    password {password}
    Parameter DEMO:

    /oauth/token?client_id=e71ae1174f2840f197eb15e6aedbcfc2Zb5Qp6xE&client_secret=xuaoMCmrK5EMdTKiHNgr1miMQTSOYGqOIZrujmwy&grant_type=password&scope=read&username={username}&password={password}


    Response:

    {"access_token":"f9355f8d-110a-4e81-904e-8e1c426fdde8","token_type":"bearer","refresh_token":"b9d8a82b-39d0-43da-b665-102848d1ee9d","expires_in":43199,"scope":"read"}

[WEB SSO] Get access_token public

Web Application Login KSO

  • Request URI: /oauth/token POST
    Request Parameters:
    Param-Key Param-Value Remark
    grant_type authorization_code Fixed
    client_id {appKey}
    client_secret {appSecret}
    redirect_uri {redirect_uri} The client registered Callback-URL
    code {code} The code that from after login successful
    Parameter DEMO:

    /oauth/token?client_id=c2f833fe192b456f973933f7db96a99c0qrUNqop&client_secret=uo7q4zyYtD93unT756w8cE62dAs4qG0J1eWl2CbO&grant_type=authorization_code&code={code}&redirect_uri=http%3A%2F%2F175.25.30.7%3A8080%2Fkso-web-demo%2Foauth_callback


    Response:

    {"access_token":"f9355f8d-110a-4e81-904e-8e1c426fdde8","token_type":"bearer","refresh_token":"b9d8a82b-39d0-43da-b665-102848d1ee9d","expires_in":43199,"scope":"read"}

Refresh access_token public

Refresh the access_token, get a new access_token by refresh_token

  • Request URI: /oauth/token POST

    Request Parameters:
    Param-Key Param-Value Remark
    grant_type refresh_token Fixed
    client_id {appKey}
    client_secret {appSecret}
    refresh_token {refresh_token}
    Parameter DEMO:

    /oauth/token?client_id=a27d9be6fbbc4c0d8f3eb568fc85a6caHZ2fSJjB&client_secret=xpdV6EEhFfb0ZwC9hSWWdCXKOxhY5quMO2dqTYjo&grant_type=refresh_token&refresh_token=b36f4978-a172-4aa8-af89-60f58abe3ba1


    Response:

    {"access_token":"d3710705-e0a3-4235-9413-d5a265d79f4e","token_type":"bearer","refresh_token":"82acf505-cfb9-4c19-996f-c2505cba1224","expires_in":43199,"scope":"read"}

Reset current account password

Use the API for reset the current account password

  • Request URI:
    [APP Mobile] /api/app/reset_password POST REST
    [Web SSO] /api/web/reset_password POST REST

    Content-Type: application/json

    Request Parameters:
    Param-Key Param-Value Remark
    oldPassword {oldPassword} Account old password
    newPassword {newPassword} Account new password, length >= 8
    Parameter DEMO:

    {"oldPassword":"12345678","newPassword":"Lo5874#52"}


    Response:

    {"success":true,"errors":[]} (Reset password successful)

    {"success":false,"errors":["302"]} (Reset password has errors)

Update current account profile

Use the API for update the current account fullName, email and phoneNumber

  • Request URI:
    [APP Mobile] /api/app/update_profile POST REST
    [Web SSO] /api/web/update_profile POST REST

    Content-Type: application/json

    Request Parameters:
    Param-Key Param-Value Remark
    fullName {fullName} Account fullName, empty value will ignore
    email {email} Account email, empty value will ignore
    phoneNumber {phoneNumber} Account phoneNumber, empty value will ignore, unique
    Parameter DEMO:

    {"fullName":"张小明","email":"ming@demo.com","phoneNumber":"18965475215"}


    Response:

    {"success":true,"errors":[]} (Reset password successful)

    {"success":false,"errors":["303"]} (Reset password has errors)
